# 크롬 웹 스토어 배포 가이드

이 확장 프로그램을 Chrome Web Store에 게시하는 전체 절차입니다.
(실제 등록은 구글 계정·결제·심사가 필요하므로 사용자가 직접 진행해야 합니다.)

## 1. 사전 준비

- [ ] Google 계정
- [ ] **개발자 등록비 $5** (1회성, 계정당) 결제 수단
- [ ] 스토어 등록 자료
  - 아이콘 128×128 (이미 `icons/icon128.png` 포함)
  - **스크린샷** 1장 이상: 1280×800 또는 640×400 (요약/번역 화면 캡처 권장)
  - 짧은 설명(최대 132자) / 상세 설명
  - 개인정보 처리방침 URL (아래 4번 참고)

## 2. 패키지(zip) 만들기

프로젝트 루트에서:

```bash
./package.sh
```

`build/smart-summarize-extension.zip` 가 생성됩니다.
(README.md, DEPLOY.md, package.sh, build/ 등은 zip에서 제외됩니다.)

## 3. 개발자 대시보드 등록

1. https://chrome.google.com/webstore/devconsole 접속 후 로그인
2. 최초 1회 **$5 등록비** 결제
3. **새 항목(New Item)** → 위 zip 업로드
4. 스토어 등록정보 작성:
   - **카테고리**: 생산성(Productivity)
   - **언어**: 한국어(기본) / 필요 시 영어 추가
   - 스크린샷, 아이콘, 설명 입력
5. **개인정보 보호** 탭:
   - 권한 사유 기재 (아래 표 참고)
   - "원격 코드 사용 안 함" 선택
   - 데이터 사용 공개: "사용자 데이터를 수집/판매하지 않음" 체크

### 권한 사용 사유 (심사 시 입력)

| 권한 | 사유 |
|------|------|
| `activeTab` / `scripting` | 사용자가 버튼을 누른 **그 순간의 현재 탭에만** 추출 스크립트를 실행해 본문·선택 텍스트를 읽음 (상시 주입 없음) |
| `storage` | API 키와 사용자 설정을 브라우저에 저장 |
| `contextMenus` | 우클릭 메뉴로 선택 영역 요약/번역 제공 |
| `host_permissions`(api.openai.com, api.anthropic.com) | 사용자 키로 AI API 호출 |

## 4. 개인정보 처리방침

웹 스토어는 권한이 있는 확장에 정책 URL을 요구합니다. 핵심 문구 예시:

> 본 확장 프로그램은 별도의 서버를 운영하지 않습니다. 사용자가 입력한 API 키와
> 설정은 브라우저의 로컬/동기화 저장소에만 저장되며 개발자에게 전송되지 않습니다.
> 요약·번역을 위해 사용자가 선택한 텍스트는 사용자 본인의 API 키로 OpenAI 또는
> Anthropic에 직접 전송되며, 각 제공자의 개인정보 처리방침이 적용됩니다.

GitHub Pages, Notion 공개 페이지, Google Sites 등에 게시한 뒤 URL을 입력하면 됩니다.

## 5. 제출 & 심사

- **검토를 위해 제출** 클릭
- 심사는 보통 수 시간 ~ 수 일 소요
- 승인되면 공개 / 거부되면 사유에 따라 수정 후 재제출

## 버전 업데이트

1. `manifest.json`의 `version` 증가 (예: `1.0.0` → `1.0.1`)
2. `./package.sh`로 새 zip 생성
3. 대시보드의 기존 항목 → **새 패키지 업로드** → 제출
